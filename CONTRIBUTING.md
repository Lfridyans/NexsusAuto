# Contributing to NEXUS // SWARM

Thank you for your interest in contributing to NEXUS // SWARM! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/nexus-swarm.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`
5. Make your changes
6. Test your changes: `npm run build`
7. Commit your changes: `git commit -m 'Add some feature'`
8. Push to your branch: `git push origin feature/your-feature-name`
9. Open a Pull Request

## Code Style

- Follow the existing code style
- Use TypeScript for all new code
- Use functional React components with hooks
- Follow the project's ESLint configuration
- Write meaningful commit messages

## Pull Request Process

1. Update the README.md with details of changes if needed
2. Ensure your code builds without errors: `npm run build`
3. Test your changes locally: `npm run dev`
4. Update documentation as needed
5. Your PR will be reviewed and merged if approved

## Development Guidelines

- Keep functions small and focused
- Add comments for complex logic
- Use TypeScript types properly
- Follow React best practices
- Test your changes thoroughly

## Questions?

If you have questions, please open an issue on GitHub.
